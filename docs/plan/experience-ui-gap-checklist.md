# 时光足迹页 UI 对齐清单

> 目标：对照 `timeline-demo.html`，梳理当前 `src/app/experience/page.tsx` 在视觉与交互上的差距，作为后续 UI 对齐与迭代的执行清单。
>
> 对照基准：
> - 当前实现：`src/app/experience/page.tsx`
> - 参考 Demo：`timeline-demo.html`
> - 关联方案：`docs/plan/experience-page-refactor.md`

---

## 一、结论摘要

当前页面已经对齐了以下基础能力：

- 左侧时间轴 + 右侧内容卡片的整体信息结构
- 工作经历下挂项目标签
- 项目标签打开右侧详情抽屉
- 抽屉的基础进入/退出动画
- 基于 `--section-color` 的主题色体系

但整体仍处于「结构接近、视觉差距明显」的状态。当前与 Demo 的核心差异，主要集中在：

- 页面级视觉氛围仍偏“普通栏目页”，而 Demo 更像“单独设计的展示页”
- 卡片尺寸、留白、版心宽度尚未对齐
- 标题系统、职位标签、正文排版仍然偏弱
- 项目标签与卡片组件感不足，尚未达到 Demo 的完整 UI 语言

---

## 二、执行优先级

### P0：必须优先对齐

这些项会直接决定页面第一眼是否“像 Demo”。

1. 页面主标题文案、字号、留白、对齐方式
2. 页面整体版心宽度与卡片宽度
3. 左侧大面积蓝色背景氛围
4. 职位信息从副标题改为右上角 Badge
5. 项目标签区域结构与样式
6. 卡片体积感、圆角、阴影、毛玻璃强度
7. 公司名、正文、时间文本的字号系统

### P1：建议在主结构对齐后处理

这些项会显著提升完成度和“精修感”。

1. 时间轴节点 glow、边框和 hover 状态微调
2. 正文高亮策略从“数字高亮”升级到“数字 + 结果短语高亮”
3. 抽屉面板的内部排版、头部样式、层级关系
4. 标签 hover 动画与箭头动势
5. 时间文本与节点之间的空间比例

### P2：可选优化

这些项不影响大方向，但会增强成品感。

1. 页面级装饰点、背景柔光细节
2. 卡片 hover 的更细腻浮动与阴影变化
3. 移动端对齐 Demo 的进一步精修
4. 教育条目的独立视觉加强

---

## 三、分类清单

### A. 视觉结构调整

#### A1. 页面标题系统未完全对齐

- 优先级：`P0`
- 当前问题：
  - 当前页面标题为“学习与工作”，Demo 为“个人经历”
  - 当前副标题为“教育背景与工作经历时间线”，Demo 为“教育与工作背景”
  - 当前标题虽然已居中，但字号、字重、留白仍弱于 Demo
- 目标状态：
  - 标题成为首屏主视觉锚点，视觉重量明显高于当前版本
  - 副标题更轻、更短，和主标题形成清晰层级
- 涉及文件：
  - `src/app/experience/page.tsx`
  - 如需调整通用头部策略，可能涉及 `src/components/PageContainer.tsx`
- 风险等级：`低`

#### A2. 页面版心过窄，卡片横向空间不足

- 优先级：`P0`
- 当前问题：
  - 当前 experience 页沿用 `PageContainer` 默认 `max-w-3xl`
  - Demo 更接近 `max-w-5xl` 级别
  - 这会直接导致卡片偏窄、正文换行过早、项目标签容易折行
- 目标状态：
  - experience 页单独拥有更宽的版心
  - 左侧时间区、时间轴、右侧卡片三者比例更接近 Demo
- 涉及文件：
  - `src/components/PageContainer.tsx`
  - `src/app/experience/page.tsx`
- 风险等级：`中`
- 备注：
  - 该项会影响整个 experience 页的横向布局，是后续多项 UI 对齐的前置条件

#### A3. 页面背景氛围缺失

- 优先级：`P0`
- 当前问题：
  - 当前背景偏纯白，缺少 Demo 左侧大面积淡蓝色柔光
  - 没有背景层支持时，时间轴区域显得空、淡、薄
- 目标状态：
  - 页面左侧形成明显但不过度的蓝色背景氛围
  - 背景作为情绪层存在，不干扰正文阅读
- 涉及文件：
  - `src/app/experience/page.tsx`
  - 或 `src/app/globals.css`
- 风险等级：`低`

---

### B. 卡片组件调整

#### B1. 卡片尺寸、体积感与 Demo 差距明显

- 优先级：`P0`
- 当前问题：
  - 当前卡片偏窄、偏高，信息堆叠感较强
  - Demo 卡片更宽、更舒展，体积感更柔和
  - 当前虽然已加 glass-card，但更像普通站内卡片
- 目标状态：
  - 卡片宽度、内边距、圆角和边框更接近 Demo
  - 玻璃感更轻，阴影更柔和，边界更有空气感
- 涉及文件：
  - `src/app/globals.css`
  - `src/app/experience/page.tsx`
- 风险等级：`中`

#### B2. 公司名与正文的字号系统偏小

- 优先级：`P0`
- 当前问题：
  - 公司名不够像主标题，正文更像普通页面段落
  - 与 Demo 相比，卡片内的文字整体显得“信息化”而不是“展示化”
- 目标状态：
  - 公司名更大、更粗
  - 正文更易扫读，阅读节奏更舒展
  - 高亮数字在更大的字号系统中形成自然重点
- 涉及文件：
  - `src/app/experience/page.tsx`
- 风险等级：`低`

#### B3. 职位信息位置错误

- 优先级：`P0`
- 当前问题：
  - 当前职位仍在公司名下方，以副标题形式存在
  - Demo 中职位在右上角，以图标 + 胶囊 badge 的方式展示
- 目标状态：
  - 公司名与职位形成左右平衡的标题区
  - 职位从“普通文字”升级为“结构化身份标签”
- 涉及文件：
  - `src/app/experience/page.tsx`
- 风险等级：`低`

#### B4. 正文高亮策略还不够完整

- 优先级：`P1`
- 当前问题：
  - 目前主要只做了数字高亮
  - Demo 中更像是“数字 + 结果导向内容”共同形成视觉节奏
- 目标状态：
  - 数字维持主题色高亮
  - 关键结果语句可以获得二级强调
- 涉及文件：
  - `src/app/experience/page.tsx`
  - 可能涉及 `src/constants/profile.ts`（如引入轻量标记）
- 风险等级：`中`

---

### C. 标签与抽屉调整

#### C1. 项目标签区域结构未完全对齐 Demo

- 优先级：`P0`
- 当前问题：
  - 当前仍保留“代表项目”标题行
  - Demo 截图中更接近“正文后直接接项目入口标签”
- 目标状态：
  - 项目标签区更像卡片内部的操作入口层
  - 信息层级更加简洁，不额外插入模块标题打断节奏
- 涉及文件：
  - `src/app/experience/page.tsx`
- 风险等级：`低`

#### C2. 项目标签按钮感不足

- 优先级：`P0`
- 当前问题：
  - 当前标签更像普通 chip
  - Demo 标签更宽、更高，内部有小圆点和右箭头，具备明显进入详情的 affordance
- 目标状态：
  - 标签本身成为卡片中的“次级 CTA”
  - 一眼可识别为可点击入口
- 涉及文件：
  - `src/app/experience/page.tsx`
  - `src/app/globals.css`
- 风险等级：`低`

#### C3. 标签布局规整性不足

- 优先级：`P1`
- 当前问题：
  - 当前标签容易因为卡片宽度不足而断成不自然的行
  - Demo 中标签排布更完整、更平衡
- 目标状态：
  - 通过卡片宽度、标签宽度与 gap 调整，让标签区更工整
- 涉及文件：
  - `src/app/experience/page.tsx`
- 风险等级：`低`

#### C4. 抽屉动画方向已对齐，但内部 UI 仍偏站内侧栏

- 优先级：`P1`
- 当前问题：
  - 当前抽屉已有右侧滑入和背景渐暗
  - 但抽屉内部头部、层级和排版还不够像 Demo 风格
- 目标状态：
  - 抽屉内部更像“详情展示面板”，而不是通用侧边栏
  - 头部、时间胶囊、内容区间距更精致
- 涉及文件：
  - `src/app/experience/page.tsx`
  - `src/app/globals.css`
- 风险等级：`中`

---

### D. 时间轴与信息层级调整

#### D1. 时间文本与时间轴节点的比例仍需校准

- 优先级：`P0`
- 当前问题：
  - 当前时间已经放到左侧，但字体气质和位置关系仍未完全像 Demo
  - 时间与节点之间的距离感仍偏“排版结果”，而不是“设计结构”
- 目标状态：
  - 时间文本成为稳定的左侧信息列
  - 与时间轴线、节点、卡片形成更完整的横向秩序
- 涉及文件：
  - `src/app/experience/page.tsx`
- 风险等级：`低`

#### D2. 时间轴节点的 glow、边框、悬浮反馈仍需精修

- 优先级：`P1`
- 当前问题：
  - 当前 dot glow 已有基础效果
  - 但边框、光晕、图标之间的关系还不够像 Demo
- 目标状态：
  - Dot 具备更柔和、统一的焦点感
  - hover 时的放大与 glow 变化更自然
- 涉及文件：
  - `src/app/globals.css`
  - `src/app/experience/page.tsx`
- 风险等级：`低`

#### D3. 条目间距与首屏纵向节奏仍偏紧

- 优先级：`P1`
- 当前问题：
  - 标题、第一条经历、第二条经历之间的留白层级还不够像 Demo
  - 当前更像信息页，Demo 更像展示页
- 目标状态：
  - 时间轴条目之间形成稳定节奏
  - 首屏更有呼吸感
- 涉及文件：
  - `src/app/experience/page.tsx`
- 风险等级：`低`

- 优先级：`P2`
- 当前问题：
  - 当前动效已经存在，但还没有达到 Demo 那种“自然轻盈”的感觉
- 目标状态：
  - hover 更细腻，动效存在但不抢戏
- 涉及文件：
  - `src/app/globals.css`
  - `src/app/experience/page.tsx`
- 风险等级：`低`

## 四、建议实施顺序

### 第一轮：先解决“结构不像”的问题

1. 标题文案、字号、留白
2. 页面版心放宽
3. 左侧背景氛围
4. 职位 badge 化
5. 时间文本与卡片横向比例调整

### 第二轮：解决“卡片不像”的问题

1. 卡片宽度、内边距、圆角、阴影
2. 公司名与正文字号系统
3. 正文高亮节奏

### 第三轮：解决“标签与交互不像”的问题

1. 去掉“代表项目”标题行
2. 项目标签对齐 Demo 结构
3. 抽屉内部 UI 和进入/退出节奏精修

### 第四轮：补完成品感

1. 背景装饰点与细节柔光
2. hover 微动效
3. 教育条目一致性收尾

  - 仅在需要对“结果类短语”做手工强调时才可能调整

## 六、建议的下一步动作

如果继续推进 UI 对齐，建议不要再做零散微调，而是按下面方式执行：

1. 先完成 `P0` 项，确保页面整体轮廓真正接近 Demo
2. 再处理 `P1` 项，提升卡片与交互完成度
3. 最后视时间处理 `P2` 项，增强成品感

换句话说，下一轮开发应聚焦“版心 + 背景 + 标题 + badge + 标签结构”这五个核心问题，而不是继续先做局部动画或抽屉细节。
