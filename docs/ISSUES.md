### #001 - 主题切换状态不同步
**发现日期**: 2026-03-06  
**状态**: 🟢 已修复  
**优先级**: 高  
**描述**: 当进入子页面并返回首页时，页面依旧是夜间模式，但切换按钮却变回日间状态（Sun 图标），状态不同步。  
**原因分析**: `useSideNavState` hook 使用了本地 `useState` 管理 `isDark` 状态，而不是使用 `next-themes` 的全局状态，导致页面切换时状态丢失。  
**解决方案**: 
- 在 `useSideNavState.ts` 中引入 `next-themes` 的 `useTheme` hook
- 使用 `resolvedTheme` 判断当前主题（`resolvedTheme === "dark"`）
- 使用 `setTheme` 切换主题，替代手动操作 DOM 的 `classList.toggle`

**修复日期**: 2026-03-06

### #002 - 夜间模式下首页气泡窗文字不可见
**发现日期**: 2026-03-06  
**状态**: 🟢 已修复  
**优先级**: 高  
**描述**: 夜间模式下首页的气泡窗（Amber 卡片）文字颜色与背景色对比度不足，导致文字不能正常显示。  
**原因分析**: Amber (#FFB800) 是亮黄色背景，在暗色模式下需要使用深色文字才能有足够对比度。  
**解决方案**: 
- 使用纯黑色和深棕黑色
- `--card-amber-ink`: `#000000`（纯黑）
- `--card-amber-text`: `#1a1200`（深棕黑）
- `--card-amber-text-muted`: `#3d2f00`（中等深度棕黑）

**修复日期**: 2026-03-06

### #003 - 导航栏 tooltip 气泡窗在暗色模式下文字不清晰
**发现日期**: 2026-03-06  
**状态**: 🟢 已修复  
**优先级**: 高  
**描述**: 导航栏图标悬浮时的 tooltip 气泡窗，背景色和文字颜色在暗色模式下对比度不足，文字看不清。  
**原因分析**: tooltip 文字颜色使用 `NAV_STYLES.tooltipText`（即 `--on-color-text`，白色），而背景色使用各图标的板块色。在暗色模式下，某些板块色（如主页的 `--foreground`）接近白色，导致白色文字在浅色背景上看不清。  
**解决方案**: 
- 将 tooltip 文字颜色改为 `#000000`（纯黑色）
- 添加 `shadow-lg` 增强视觉层次
- 背景色保持使用各图标的板块色（Amber、Indigo、Sky、Emerald 等高饱和色）
- 黑色文字在所有高饱和板块色上都有足够对比度

**修复日期**: 2026-03-06

### #004 - 导航栏 tooltip 气泡窗统一白色字体 + 主页底色对比度修复
**发现日期**: 2026-03-06  
**状态**: 🟢 已修复  
**优先级**: 高  
**描述**: 用户要求所有导航栏 tooltip 统一使用白色字体；主页 tooltip 在夜间模式下底色 (`--foreground` = `#f8fafc` 近白色) 与白色文字对比度不足，需要切换底色保证可见性。  
**原因分析**: 主页导航色直接使用 `var(--foreground)`，夜间模式下为 `#f8fafc`（近白色），无论文字用黑色还是白色，在浅色底上的 tooltip 视觉效果都不理想。需要单独为导航主页色设定日夜不同的值。  
**解决方案**: 
- 新增 CSS 变量 `--nav-home`：日间 `#0f172a`（深色底+白字），夜间 `#1e293b`（深色底+白字）
- `NAV_STYLES.colors.home` 从 `var(--foreground)` 改为 `var(--nav-home)`
- `NAV_STYLES.tooltipText` 从 `var(--on-color-text)` / `#000000` 统一改为 `#ffffff`
- 所有板块色（Amber/Indigo/Sky/Emerald）配白色文字均有足够对比度

**修复日期**: 2026-03-06
