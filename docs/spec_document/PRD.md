# PRD.md — 产品需求文档

> 本文件是 **webjianli** 项目的唯一合同与规格说明书。所有开发决策必须以此为依据。

---

## 一、项目概述

| 项目 | 详情 |
|------|------|
| **产品名称** | Pardon 乌冬面 · 个人品牌空间 |
| **产品形态** | 静态个人网站（无后端、无数据库） |
| **构建目标** | 为彭焕东（英文名 Pardon）打造一个具有强烈视觉辨识度的在线简历 + 个人品牌主页 |
| **核心受众** | ① HR / 招聘官（求职场景）；② 行业同行 / 社区（品牌场景） |
| **核心价值** | 在 5 秒内建立"懂 AI 产品 + 能落地 + 有品味"的第一印象 |
| **部署平台** | Vercel（使用 Vercel 提供的 `.vercel.app` 子域名） |

## 二、核心功能（In-Scope）

### 2.1 首页（`/`）— Bento Grid 导航中枢
- 全屏 Bento Grid 布局，4 张功能卡片导航至各子页面
- 每张卡片拥有独立板块色彩：Amber / Indigo / Sky / Emerald
- ParticleTrail 鼠标粒子拖尾特效
- FloatingShapes 浮动几何装饰（仅 About 卡片）
- 全站导航栏（详见 §2.9）
- MagneticButton 磁力悬停按钮
- TiltCard 3D 倾斜效果（仅 Projects 卡片）
- 进场动画（Framer Motion stagger 入场）

### 2.2 个人介绍页（`/about`）— 板块色 Amber `#FFB800`
- 座右铭横幅（大号衬线字体 + 左侧 Amber 色竖线）
- 姓名 + 定位标题（衬线超大字 + Amber 色点缀）
- 个人摘要段落
- 工具栈标签组（hover 变 Amber 色）
- 能力雷达：4 块能力卡片，每块有独立色点（靛蓝/绿/粉/Amber）
- MBTI 特质展示（`ENFJ` 大号 Amber 色等宽字体）
- 整页背景、选中色、强调色全部跟随 Amber 板块色

### 2.3 学习与工作页（`/experience`）— 板块色 Sky Blue `#0EA5E9`
- 时间线布局（竖线 + 节点图标）
- 工作经历卡片 + 教育背景卡片（按时间降序）
- 节点图标：Building2（工作）/ GraduationCap（教育）
- 荣誉列表（圆点列表）
- 整页背景、时间线色、强调色全部跟随 Sky Blue 板块色

### 2.4 项目经历页（`/projects`）— 板块色 Indigo `#4800ff`
- 每个项目独立 Article 卡片（溢出隐藏，圆角）
- 项目头部：名称 + 副标题 + 外链（GitHub / 在线体验 / 演示视频）
- 元信息：团队规模 + 时间周期
- 项目描述段落
- 我的角色标注
- 数据指标格子（Stats Grid，2×2 或 1×4）
- 核心成果列表
- 技术栈标签组
- 整页背景、强调色全部跟随 Indigo 板块色

### 2.5 日常思考页（`/thoughts`）— 板块色 Emerald `#10B981`
- 文章卡片列表（日期 + 标题 + 摘要 + 标签）
- 标签统一使用 Emerald 板块色（不做多色化）
- 底部"持续更新"占位提示
- 整页背景、标签色、强调色全部跟随 Emerald 板块色

### 2.6 暗色模式
- 使用 `next-themes` 实现系统级 / 手动主题切换
- 主题切换按钮集成在全站导航栏中（Sun/Moon 图标；桌面端 tooltip 显示“切换主题”，移动端仅图标）
- 防白闪：在 `<html>` 上通过 `next-themes` 注入主题类名
- 暗色模式下所有 4 个板块色保持不变（高饱和色在暗色背景下更突出）

### 2.7 全站设计系统（buildaloud 风格移植）
- **板块色彩身份系统**：每个子页面的 `background`、`selection`、`accent`、`border-hover`、`link`、`timeline` 等全部使用该板块主色
- **文字选中色跟随板块**：`::selection` 背景色在各子页面覆盖为对应板块色
- **阴影策略**：内容卡片不使用 `box-shadow`，层级靠颜色和边框区分；导航栏和特殊视觉效果组件（ParticleTrail、FloatingShapes 等）允许保留阴影
- **衬线字体张力**：标题使用 Noto Serif SC，正文使用 Geist Sans
- **OkLab 色彩混合**：CSS 中 hover 态半透明叠色使用 `color-mix(in oklab, ...)`；Canvas/SVG 等特效中保留 `rgba()` 但统一定义为常量
- **颜色常量化**：全站主站功能的所有颜色值统一来自 `src/constants/theme.ts`，由 `layout.tsx` 注入 `:root/.dark` 变量；`globals.css` 不写死 Hex（`/playground` 和 Canvas 特效的动态计算除外）

### 2.8 响应式设计
- Mobile（< 768px）、Tablet（768px–1023px）、Desktop（≥ 1024px）三档适配
- 首页 Bento Grid 在 Mobile 退化为单列堆叠
- 导航栏：Desktop 桌面端左侧垂直悬浮；Mobile 移动端底部水平固定

### 2.9 全站导航栏
- **桌面端（≥ 768px）**：左侧垂直悬浮导航（现有 SideNav），fixed 定位，始终可见
- **移动端（< 768px）**：底部水平固定导航栏，fixed 定位，始终可见
- **导航项**：主页 / 个人信息 / 灵感造物 / 时光足迹 / 思想碎片 / 切换主题（共 6 项）
- **显示形式**：仅图标（桌面端 hover 显示 tooltip，移动端不显示文字）
- **当前路由高亮**：当前页面对应的导航图标使用该板块色高亮显示
- **主题切换**：Sun/Moon 图标 + 桌面端 tooltip“切换主题”，集成在导航栏最后一项，使用 `next-themes` 的 `useTheme` hook
- **阴影**：导航栏允许保留 `box-shadow`（例外于内容卡片的零阴影规则）
- **无 BackButton**：全站导航已覆盖返回首页功能，不再需要 BackButton 组件

## 三、范围外（Out-of-Scope）

| 功能 | 排除原因 |
|------|----------|
| 后端 / 数据库 / CMS | 内容静态，无动态更新需求 |
| 用户认证（登录/注册） | 公开只读网站 |
| 联系表单 / 访客留言 | 无后端，联系方式已在首页直接展示 |
| `/playground` 页面 | 与求职主题无关，排除在规划外 |
| 自定义域名 | 暂时使用 Vercel 子域名 |
| 多语言国际化（i18n） | 首期全中文，英文仅用于标签/代号 |
| 博客 CMS 后台管理 | thoughts 页内容 hardcode 在 constants/profile.ts |
| SEO 深度优化 | 首期不作精细 SEO 优化 |
| 性能分析埋点 | 无 Analytics 需求 |

## 四、用户故事

### Story 1：HR 快速筛查
> 「作为一名招聘 AI 产品经理的 HR，我希望打开链接后在 5 秒内看到候选人的核心定位和项目亮点，而不需要阅读冗长的 PDF 简历。」

**验收标准**：首页 Bento Grid 在 5 秒内完成动画入场，4 个卡片清晰传达 4 个维度（个人/项目/经历/思考）。

### Story 2：面试官深度了解
> 「作为面试官，我希望点击"项目经历"后看到有数据支撑的成果（用户数、游戏场次等），以评估候选人的实际能力。」

**验收标准**：`/projects` 页面的 Stats Grid 清晰展示核心数字，数据来自 `constants/profile.ts`。

### Story 3：夜间浏览舒适度
> 「作为一个在夜间查看候选人资料的技术面试官，我希望网站支持暗色模式，不伤眼。」

**验收标准**：暗色模式开关可用，切换无白闪，所有板块色在暗色背景下清晰可读。

### Story 4：手机端快速浏览
> 「作为一名在手机上收到简历链接的 HR，我希望网站在手机端可以正常浏览，布局不错乱。」

**验收标准**：Mobile 端（375px 宽）所有页面正常渲染，Bento Grid 退化为单列，无横向溢出。

## 五、成功标准（Definition of Done）

| 验收项 | 标准 |
|--------|------|
| 首页动画 | Bento Grid 4 张卡片按序 stagger 入场，总时长 ≤ 1.2s |
| 板块色彩身份 | 4 个子页面各自的 accent、selection、timeline/border 颜色 = 对应板块色 |
| 暗色模式 | `next-themes` 可切换，无白闪，暗色下配色正确 |
| 响应式 | 375px / 768px / 1280px 三档无布局断裂 |
| Vercel 部署 | `next build` 无报错，部署后 URL 可访问 |
| 静态内容同步 | 所有展示数据来自 `src/constants/profile.ts`，无硬编码重复 |
| 颜色常量化 | 主站功能的所有颜色值来自 `src/constants/theme.ts`，无 Hex 硬编码 |
| 内容卡片零阴影 | 内容区卡片不使用 `box-shadow`（导航栏和特效除外） |
| OkLab hover | CSS 中的卡片 hover 态使用 `color-mix(in oklab, ...)`（Canvas/SVG 特效除外） |
| 选中色 | 每个子页面 `::selection` 背景色 = 该页面板块色 |
| 全站导航 | 桌面侧边 + 移动端底部固定，当前路由高亮，主题切换集成 |

## 六、非目标（Anti-Goals）

- **不做功能堆砌**：不因为"可以加"就加功能，每个元素必须服务于求职或品牌目标
- **不做过度动画**：动画服务于感知，不炫技；所有动画 `duration ≤ 0.8s`
- **不做复杂状态管理**：无需 Redux / Jotai / Zustand，纯 React state 即可
- **不做后端依赖**：任何需要服务器的功能均不在范围内
